## Round 8 Fixes (Bounded Correction)

This round addresses blocking findings from reviewer visit 7:

1. **R5-R7: Announce-only custom compatibility fix** (`useDataTable.ts`, `usePivotTable.ts`):
   - Changed initial `createDataTable/PivotTable` to always pass the channel, not the minimal announcer object
   - Previously, the hook passed `options.announcer ?? announcerChannelRef.current` which meant a minimal `{ announce }` object would bypass the channel subscription
   - Now always passes `announcerChannelRef.current!` to ensure the live region receives all announcements

2. **R2-R7: Source version lifecycle fix** (`createClientDataSource.ts`):
   - Changed to resolve `getVersion` function from `DataVersion<TRow>` config
   - Previously only published `opts.dataVersion?.version` and dropped `getVersion`
   - Now derives token from rows when `getVersion` is provided: `opts.dataVersion.getVersion(rows)`

3. **R4-R7: Pivot version contract fix** (`packages/pivot/src/types.ts`):
   - Changed `PivotTableOptions.dataVersion` from `string | number` to `DataVersion<TRow>`
   - Now uses the shared type alias across core/client/hook boundaries
   - Re-exported `DataVersion` type from pivot package

4. **R3-R7: Nullable source cursor ownership fix** (`useDataSource.ts`):
   - Added `cursorSelectionRef.current = { cursor: null, direction: 'next' }` in null-source branch
   - Previously source removal did not clear owned cursor state
   - Now explicitly resets cursor selection when source is removed

5. **R7: Evidence accuracy fix** (this document):
   - Corrected HEAD SHA distinction between implementation (39811c5) and evidence commits
   - Updated test counts to match current verification output
   - Added new test coverage for R5 minimal announcer compatibility

6. **Test coverage added** (`multi-instance-announcer.test.tsx`):
   - Added two new tests for minimal `{ announce }` announcer compatibility:
     - `R5-R7: minimal { announce } object reaches live region via channel`
