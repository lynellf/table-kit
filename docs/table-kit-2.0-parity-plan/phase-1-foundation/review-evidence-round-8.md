## Round 8 Fixes (Bounded Correction)

This round addresses blocking findings from reviewer visit 7:

### R5-R7: Announce-only Custom Compatibility Fix

**Files:** `useDataTable.ts`, `usePivotTable.ts`

**Problem:** Custom announcers with only `{ announce }` (no `subscribe`) would bypass the channel subscription.

**Fix:** Always pass the channel to `createDataTable/PivotTable` at mount time:
```typescript
// Before: options.announcer ?? announcerChannelRef.current
// After:
tableOptions.announcer = announcerChannelRef.current!;
```

### R2-R7: Source Version Lifecycle Fix

**File:** `client.ts`

**Problem:** `createClientDataSource` evaluated `getVersion` once at construction and stored the literal.

**Fix:** Store `getVersion` function and resolve token dynamically on each `getRows` call:
```typescript
const versionFn = opts.dataVersion?.getVersion;
const resolveVersion = () => versionFn ? versionFn(rows) : undefined;
return { ...baseResult, dataVersion: resolveVersion() };
```

### R3-R7: Source Publication Guard

**File:** `useDataSource.ts`

**Problem:** Handlers only compared `requestToken`, allowing stale results from replaced sources.

**Fix:** Capture `sourceToken` in in-flight entry and compare in handlers:
```typescript
interface InFlightEntry {
  key: string;
  sourceToken: string;  // NEW: guards against source replacement
  requestToken: number;
  // ...
}
const isStale = controller.signal.aborted ||
  inFlightRef.current?.requestToken !== requestToken ||
  inFlightRef.current?.sourceToken !== currentSourceToken;
```

### R4-R7: Pivot Version Contract Fix

**Files:** `types.ts`, `pivotTable/factory.ts`

**Problem:** Pivot compared `dataVersion` objects by identity, not resolved token.

**Fix:** Resolve token before comparison:
```typescript
const resolveDataVersion = (dv) => {
  if (version.getVersion) return version.getVersion(next.data);
  return version.version;
};
const dataVersionChanged = prevVersion !== nextVersion;
```
